🏠 ¿Qué es Hermes Agent?

🧠 ¿Qué es Hermes Agent?

Hermes Agent es un framework de agente de IA open-source desarrollado por Nous Research. Se ejecuta en tu terminal, plataformas de mensajería e IDEs. Pertenece a la misma categoría que Claude Code, Codex y OpenClaw — agentes autónomos que usan llamadas a herramientas para interactuar con tu sistema.


¿Qué lo hace diferente?

  • 🧩 Auto-mejora mediante skills — aprende de la experiencia guardando procedimientos reutilizables como skills.
  • 🧠 Memoria persistente entre sesiones — recuerda quién eres, tus preferencias y lecciones aprendidas.
  • 🌐 Gateway multi-plataforma — el mismo agente funciona en Telegram, Discord, WhatsApp, Slack y más.
  • 🔄 Provider-agnóstico — cambia de modelo y proveedor sin modificar nada más.
  • 👥 Perfiles — ejecuta instancias independientes con configuraciones aisladas.
  • 🔌 Extensible — plugins, servidores MCP, herramientas personalizadas, webhooks, cron.

🔑 Conceptos Clave

📦 Skills ▼
Procedimientos reutilizables que el agente carga para tareas especializadas. Se guardan como archivos SKILL.md con metadatos YAML.
🧠 Memoria ▼
Datos persistentes entre sesiones. Dos almacenes: memory (notas personales) y user (perfil del usuario). Se inyectan en cada turno.
💾 Sesiones ▼
Conversaciones almacenadas en SQLite con búsqueda FTS5. Se pueden reanudar, exportar, comprimir y buscar.
🔌 Providers ▼
20+ proveedores: OpenRouter, Anthropic, OpenAI, DeepSeek, Google Gemini, xAI/Grok, HuggingFace, Xiaomi MiMo, Ollama, y más. Cambia con hermes model.

📦 Skills

🧩 ¿Cómo funcionan las Skills?

Las skills son procedimientos reutilizables que Hermes carga cuando necesita hacer una tarea específica. Se guardan como archivos SKILL.md con metadatos YAML (frontmatter) y pueden incluir scripts, plantillas y referencias.


¿Para qué sirven?

  • 🧠 Memoria procedural — el agente recuerda cómo hacer tareas complejas
  • 🚀 Workflows reutilizables — desde diseño de guías educativas hasta debugging
  • 🔄 Auto-mejora — si encuentras un error en una skill, el agente la parchea

🔍 Ver skills disponibles

# Buscar en el hub de skills

hermes skills uninstall NOMBRE

🔌 MCP (Model Context Protocol)

🔌 ¿Qué es MCP?

MCP (Model Context Protocol) es un protocolo que permite conectar servidores externos de herramientas a Hermes. Es como instalar "apps" que el agente puede usar. Hermes tiene un cliente MCP nativo integrado.

⚙️ Configuración

Los servidores MCP se configuran en config.yaml bajo la sección mcp_servers:

# config.yaml
mcp_servers:
  my-server:
    transport: stdio        # o 'http'
    command: python         # para stdio
    args: ["-m", "mcp_server"]
    url: "http://..."       # para HTTP
    headers:
      Authorization: "Bearer TOKEN"

📋 Comandos MCP

hermes mcp list           Servidores configurados
hermes mcp add NOMBRE     Añadir servidor
hermes mcp remove NOMBRE  Eliminar servidor
hermes mcp test NOMBRE    Probar conexión
hermes mcp configure NOM  Activar/desactivar tools
hermes mcp serve          Ejecutar Hermes como MCP server

🔍 Convención de nombres

Las herramientas de servidores MCP aparecen con el prefijo del servidor:

mcp_mi_servidor_mi_herramienta
    ↑    ↑           ↑
  prefijo servidor  herramienta

🌐 Gateway — Plataformas de Mensajería

🌐 ¿Qué es el Gateway?

El Gateway permite que Hermes funcione en múltiples plataformas de mensajería simultáneamente — Telegram, Discord, WhatsApp, Slack, Signal, Email, SMS, Matrix, y más. Una vez configurado, puedes hablar con Hermes desde cualquier plataforma.

🧠 Sistema de Memoria

La memoria persiste entre sesiones. Hay dos almacenes:

  • memory — Notas personales del agente
  • user — Perfil del usuario

# Operaciones de memoria (batch)
memory(action='add',
       target='memory|user',
       content='...')

# Búsqueda en sesiones pasadas
session_search(query="tema",
               limit=5)

🔍 Búsqueda FTS5

Las sesiones se almacenan en SQLite con búsqueda de texto completo (FTS5). Puedes buscar en conversaciones pasadas con session_search. Cuatro modos:

  • Discovery — session_search(query="...") busca y devuelve sesiones relevantes
  • Scroll — session_search(session_id="...", around_message_id=N) navega dentro de una sesión
  • Read — session_search(session_id="...") lee sesión completa
  • Browse — session_search() sin args muestra sesiones recientes

Los perfiles permiten ejecutar instancias independientes de Hermes con configuraciones, sesiones, skills y memoria aisladas. Cada perfil tiene su propio directorio ~/.hermes/profiles/<nombre>/.

🔒 Seguridad & Privacidad

🔴 Redacción de Secretos

Activado por defecto. El output de herramientas (terminal, read_file, contenido web) se escanea para ocultar API keys, tokens y secretos antes de que entren al contexto de la conversación.

# Desactivar (solo para debugging)
hermes config set security.redact_secrets false

⚠️ Se necesita reiniciar la sesión.

🛡️ Aprobación de Comandos

Hermes pide confirmación antes de ejecutar comandos peligrosos (rm -rf, git reset --hard, etc.).

hermes config set approvals.mode manual  # Siempre preguntar
hermes config set approvals.mode smart   # Auto-aprobar seguros
hermes config set approvals.mode off     # Sin aprobación

# O en una sola ejecución
hermes --yolo ...

👤 Redacción de PII

Cuando está activado, el gateway hashea IDs de usuario y elimina números de teléfono del contexto antes de que lleguen al modelo.

hermes config set privacy.redact_pii true

🔗 Shell Hooks Allowlist

Las integraciones de shell hooks requieren aprobación explícita antes de ejecutarse. Gestionado en ~/.hermes/shell-hooks-allowlist.json.
